PROJECT FRONTIER — CLEAN START

This package is deliberately simple. Every website file is in ONE folder.
There are no nested css, js, data, or supabase folders.

PHASE 1 — GITHUB
1. Create a brand-new PRIVATE repository named project-frontier.
2. Do not add a README, .gitignore, or license during creation.
3. Open the empty repository and choose “uploading an existing file.”
4. Extract this ZIP on your computer.
5. Open the extracted project-frontier-clean-start folder.
6. Upload every file inside that folder to GitHub. Do not upload the ZIP itself.
7. Commit directly to main.

PHASE 2 — CLOUDFLARE PAGES
1. Create a new Pages project by connecting GitHub.
2. Select the new project-frontier repository.
3. Production branch: main
4. Framework preset: None
5. Root directory: leave blank
6. Build command: leave blank
7. Build output directory: .
8. Deploy command: leave blank / not required
9. Deploy and open the site.
10. Confirm that you see the Project Frontier book cover.

STOP AFTER PHASE 2.
Do not create Supabase until the website is visibly working.

PHASE 3 — SUPABASE
Only begin after Cloudflare works. Follow SUPABASE-NEXT.txt.

The app currently saves only in the browser. Use Export Data for backups.
